# 表单验证测试

对目标页面中的表单进行全面验证测试，发现并修复验证逻辑、错误提示、交互行为的问题。

## 使用方式

```bash
/form-test localhost:3000/register
/form-test localhost:3000/checkout
```

## 测试矩阵

### 必填验证
| 场景 | 操作 | 期望 |
|------|------|------|
| 全部为空 | 不填任何字段，直接提交 | 所有必填字段显示错误提示，表单不提交 |
| 部分为空 | 只填部分字段 | 仅空字段显示提示，已填字段不受影响 |
| 全空格 | 填写空格字符 | 空格应被视为空值 |

### 格式验证
| 场景 | 操作 | 期望 |
|------|------|------|
| 邮箱格式 | `notanemail`、`a@`、`@b.com` | 提示"请输入有效的邮箱地址" |
| 手机号 | `123`、`abc`、`138001380001` | 按规则校验位数和格式 |
| 密码强度 | `123`、`password`、纯数字 | 提示密码强度不足，要求字母+数字+特殊字符 |
| URL 格式 | `not-a-url` | 提示输入合法的 URL |

### 边界值
| 场景 | 操作 | 期望 |
|------|------|------|
| 最小长度 | 输入恰好等于 minLength | 通过验证 |
| 最大长度 | 输入恰好等于 maxLength | 通过验证 |
| 超出长度 | 输入超过 maxLength | 输入框限制或提示超长 |
| 金额为 0 | 输入 0 | 按业务规则处理 |
| 金额为负 | 输入 -100 | 拒绝或提示 |

### 特殊输入
| 场景 | 操作 | 期望 |
|------|------|------|
| 特殊字符 | `"><script>alert(1)</script>` | 转义处理，不触发 XSS |
| Emoji | 输入 emoji | 按业务规则处理（允许/拒绝） |
| SQL 注入 | `' OR '1'='1` | 无害化处理，不影响数据库 |
| 超长文本 | 粘贴 10000 字符 | 截断或提示 |
| 换行/tab | 输入换行符和制表符 | 正确处理 |

### 交互行为
| 场景 | 操作 | 期望 |
|------|------|------|
| 实时校验 | 输入过程中 | 失焦后触发校验，非每次按键 |
| 错误清除 | 修正错误字段 | 错误提示消失 |
| 重复提交 | 快速双击提交按钮 | 按钮置灰，防止重复提交 |
| 网络异常 | 提交时断网 | 显示网络错误提示，提供重试 |
| 成功后跳转 | 提交成功 | 正确跳转或显示成功状态 |

## 工作流程

1. **识别表单** — 打开页面，找到所有表单及其字段
2. **按矩阵测试** — 逐行执行测试矩阵中的场景
3. **记录问题** — 每个失败场景记录：输入值、期望行为、实际行为
4. **修复代码** — 定位前端验证逻辑和错误提示组件，修复
5. **回归验证** — 刷新页面重跑失败场景
6. **输出报告** — 测试通过/失败统计，已修复项列表
